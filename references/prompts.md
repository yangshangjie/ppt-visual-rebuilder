# PPT Visual Rebuilder Prompt Templates

Use these templates as copy-ready prompts. Fill bracketed fields before use. Keep the approval gates from `SKILL.md`: do not move to the next stage until the user confirms.

## 1. GPT: Organize Raw Materials Into A PPT Framework

```text
请基于我提供的资料，帮我生成一份 PPT 框架。

资料类型包括：[需求/方案文档/会议纪要/产品资料/销售材料/截图/表格/其他]
目标受众：[客户/老板/销售团队/高管/投资人/内部团队]
汇报目标：[说明方案/争取立项/销售转化/项目复盘/培训宣讲/其他]

输出要求：
1. 整体汇报逻辑
2. 章节结构
3. 每页标题
4. 每页核心观点
5. 每页正文要点
6. 每页建议呈现方式
7. 哪些页面需要图表、流程图、产品图或截图

当前阶段只生成 PPT 框架，不要生成 PPT 文件。
```

## 2. GPT: Optimize A Hand-Made PPT Prototype

```text
请阅读我这份手搓 PPT / 逐页截图，帮我从商业汇报角度检查：

1. 整体叙事逻辑是否顺畅
2. 章节顺序是否合理
3. 每页标题是否准确有力
4. 每页核心观点是否清楚
5. 每页信息量是否合适
6. 哪些页面需要合并、拆分或重写
7. 哪些页面需要强化视觉表达
8. 哪些关键数字、术语或产品名需要保持不变

请输出优化后的逐页大纲。
暂时不要生成 PPT 文件。
```

## 3. Codex: Reverse-Engineer PPT Outline

```text
$ppt-visual-rebuilder
请阅读我上传的手搓 PPT，反推完整 PPT 大纲和逐页内容。

每页请包含：
1. 标题
2. 核心观点
3. 正文要点
4. 视觉建议
5. 需要保留的关键文字、数字、产品名或截图
6. 无法识别、需要我确认的内容

当前阶段不要生成 PPT 文件，也不要生成视觉拼图。
请先输出大纲，等我确认后再继续。
```

## 4. User Approval After Outline

```text
大纲确认。
请基于这版大纲生成 3 版不同视觉风格的 PPT 拼图方案。
当前阶段只生成拼图，不要生成逐页视觉稿，不要生成 PPTX。
```

## 5. Generate Three Full-Deck Mosaics

```text
请基于已确认的 PPT 大纲和逐页内容，调用 $imagegen 生成 3 版不同视觉风格的 PPT 拼图方案。

任务目标：
请根据这套 PPT 的主题、行业属性、内容密度、受众场景和叙事风格，自行选择 3 种最适合的视觉方向，并分别生成一张完整拼图，方便我比较整体风格、页面节奏和高级感。

输出要求：
1. 生成 3 张 PPT 拼图。
2. 每张拼图都必须包含整套 PPT 的所有页面缩略图，并保持正确页序。
3. 每个缩略页都应是 16:9 横版 PPT 页面。
4. 三版风格必须明显不同，不要只是换颜色。
5. 每一版内部必须使用统一视觉系统，包括字体层级、色彩系统、背景风格、图表样式、图标风格、卡片/模块样式、页脚和页码样式。
6. 每一版都要像一套完整、正式、可落地的商业 PPT 方案，而不是零散页面草稿。
7. 拼图用于选择方向，文字可以适当缩小，但主标题、关键数字、核心图表和页面结构必须能看清。
8. 当前阶段不要生成 PPTX，不要生成逐页单图。

保存要求：
请将拼图保存到 outputs/mosaics/，命名为 mosaic-v1.png、mosaic-v2.png、mosaic-v3.png。

反幻觉要求：
不要新增未经确认的客户名称、金额、KPI 数字、业务数据、日期或服务承诺。
无法识别的内容请标记为“需用户确认”。
```

## 6. Regenerate A Mosaic Direction

```text
第 [X] 版方向接近，但还不满意。

请基于第 [X] 版重新生成一版，调整方向如下：
1. [例如：更科技、更高级、更像高管汇报、更像产品发布会]
2. [例如：加强黑银蓝、AI 中枢、数据驾驶舱、硬件主视觉]
3. [例如：减少普通商务感，减少办公照片，增强未来感和系统感]
4. [例如：必须使用我提供的产品素材 / 品牌色 / 截图作为核心视觉]

要求：
1. 仍然是完整 PPT 拼图。
2. 仍然包含所有页面，并保持正确页序。
3. 不要生成逐页单图。
4. 不要生成 PPTX。

保存要求：
请将重做后的拼图保存到 outputs/mosaics/，使用新的版本名，例如 mosaic-v4.png。
```

## 7. Expand Approved Mosaic Into Single-Page Visuals

```text
这版拼图确认。
请基于这版视觉风格，将拼图中的每一页依次扩展生成为独立的高清 16:9 单页 PPT 视觉稿。

任务目标：
这一步不是重新设计，也不是重新写内容，更不是在原图上裁剪。
而是把已选定拼图中的多页 PPT 视觉方案生成逐页高清单页图。
拼图是视觉风格、版式、内容结构和页面顺序的母版，请严格参考拼图逐页还原。

核心要求：
1. 请识别拼图中的每一页，并按拼图页码顺序依次生成单页。
2. 每次只生成一张完整单页 PPT 视觉稿，不要生成拼图，不要生成多页合成图。
3. 每张图必须是高清 16:9 横版画面，适合作为下一步可编辑 PPTX 复刻蓝本。
4. 严格沿用选定拼图的视觉系统。
5. 每一页的版式应与拼图中对应页面保持一致，但需要放大为单页高清版本，细节更清晰、更完整。
6. 主标题、核心观点、关键数字、图表标签、页码必须清晰可读。
7. 如果拼图中某些小字无法准确识别，请保持原有版式和视觉关系，并参考我已确认的 PPT 大纲/逐页内容进行补全；不要自行编造新信息。
8. 如果拼图中某些复杂图表细节无法完全识别，请按视觉关系和大致比例高清重绘，但不要凭空增加精确数据。

保存要求：
请将逐页视觉稿保存到 outputs/slides/，命名为 slide-01.png、slide-02.png 等。

请先生成第 1 页，生成后让我确认。
```

## 8. Batch Expand Single-Page Visuals

Use only if the user explicitly approves batch generation.

```text
确认批量生成全部页面。
请基于已确认拼图一次性生成全部高清 16:9 单页 PPT 视觉稿。

要求：
1. 每页单独保存为一张图片。
2. 按页码顺序命名，例如 slide-01.png、slide-02.png。
3. 不要生成拼图。
4. 不要生成 PPTX。
5. 生成完成后请统一检查封面、驾驶舱、价值页和结尾页。

保存要求：
请将全部页面保存到 outputs/slides/。
```

## 9. Rebuild One Page From A Screenshot

```text
请重做第 [X] 页。

这页必须严格按照我提供的截图模块来生成：
1. 模块数量必须一致。
2. 区域比例必须一致。
3. 图表位置必须一致。
4. 指标卡、图表卡、标题区、页脚和页码的位置关系必须一致。
5. 视觉风格仍然沿用当前 PPT 的整体风格。
6. 不要自行增加截图中没有的模块。
7. 不要凭空增加未经确认的精确数据、人名、日期、金额或百分比。

请生成一张完整高清 16:9 单页 PPT 视觉稿。
请将重做后的页面保存到 outputs/corrected/，例如 slide-10-corrected.png。
```

## 10. Fix Chinese Text

```text
这页里的「[错误词]」写错了，正确写法是「[正确词]」。

请只修正这个词/这行文字：
1. 保持整体画面不变。
2. 保持版式、背景、图片、图表、页脚和页码不变。
3. 优先使用本地字体确定性覆盖修复，不要让图片模型重新猜中文。
4. 修正后请保存为新文件，并打开检查。

保存要求：
请将修正后的页面保存到 outputs/corrected/。
```

Example:

```text
这页里的「销售」写错了，正确写法是「销售」。
请只修正这两个字，不要改整体画面。
```

## 11. Rebuild Editable PPTX

```text
所有逐页视觉稿确认。
请开始生成可编辑 PPTX。

要求：
1. 严格参考已确认的单页视觉稿复刻。
2. 尽量使用可编辑文本、形状、图表和图片，而不是整页贴图。
3. 保持页面尺寸 16:9。
4. 保持字体层级、颜色系统、页脚、页码、卡片样式、图标风格一致。
5. 不要新增未经确认的业务数据、客户名称、金额、KPI 数字或服务承诺。
6. Dashboard / 截图类页面必须严格保留模块数量、区域比例和信息层级。
7. 生成后请渲染检查，修正错字、遮挡、重叠、溢出和页码问题。

最终文件请输出到：
outputs/pptx/final-deck.pptx
```
